Passage par valeur et référence


					function multiply2(x){
						x = 2*x;
					}
					
					let value = 10;
					multiply2(value);
					console.log(value); // Affiche 10
				

					function multiply2(x){
						x[0] = 2*x[0];
					}
					
					let value = [10];
					multiply2(value);
					console.log(value[0]); // Affiche 20

Dans le code de gauche, la valeur de l'argument est passée par valeur, c'est-à-dire qu'une copie de value est faite. Modifier la copie dans la fonction ne modifie pas la valeur d'origine.

Dans le code de droite, la valeur de l'argument est passée par référence, c'est-à-dire que l'argument référence la valeur d'origine et un changement l'affectera également.

Quand est-ce que le passage est fait par valeur et quand est-il fait par référence ?

JavaScript

Passage par valeur et référence

Le passage est par valeur pour les types dit primitifs :

  • string (par exemple "Bonjour")
  • number (par exemple 42)
  • boolean (par exemple true)
  • undefined
  • null
  • bigint
  • symbol

Tous les autres types sont toujours passés par référence. Cela inclus donc tableaux, objets, promesses, etc
