Les modules

Il est bien de découper son code en plusieurs fichiers. Par exemple, on pourra définir la classe Circle dans son propre fichier, avec également une constante globale pour le cercle unitaire en (0;0). 


            // Fichier Circle.js
            class Circle{
                constructor(x, y, radius){
                    this.x = x;
                    this.y = y;
                    this.radius = radius;
                }
                area(){
                    return Math.PI*this.radius*this.radius;
                }
            }

            const UNIT_CIRCLE = new Circle(0, 0, 1);
Comment utiliser ce code dans un autre fichier ?

JavaScript

Les modules

Il serait possible d'importer tous les fichiers dans le HTML principal, à l'aide de balises <script>, mais cela a plusieurs inconvénients, en particulier

  1. Tout est importé globalement, généralement des conflits de nom. Ainsi, dans l'exemple précédent, il ne peut y avoir un autre objet ou variable se nommant Circle.
  2. Si une librairies est composée de plusieurs fichiers, ils doivent tous être importés un par un. Cela devient encombrant avec des centaines de fichiers.
  3. La gestion de dépendances est très manuelle, complexe, et sujette aux erreurs.


Il existe une solution à tout ces problèmes : Les modules

Les modules fournissent aussi, comme on va le voir plus loin, une notion d'encapsulation.

JavaScript

Les modules

Un module est un code contenu dans un fichier qui s'execute dans son contexte propre. Tout ce qui n'est pas exporté d'un module ne peut être accédé en dehors.
Pour exporter un élément, on utilise le mot clé export. 


            // Fichier Circle.js    
            export class Circle{                            // Classe exportée
                constructor(x, y, radius){
                    this.x = x;
                    this.y = y;
                    this.radius = radius;
                }
                area(){
                    return Math.PI*this.radius*this.radius;
                }
            }

            export const UNIT_CIRCLE_AREA = new Circle(0, 0, 1).area(); // Constante exportée
            const UNIT_CIRCLE_PERIMETER = 2*Math.PI;                    // Constante non exportée

JavaScript

Les modules

Pour importer un élément d'un module, on utilisera le mot-clé import


            // Fichier main.js
            import { Circle, UNIT_CIRCLE_AREA } from "./Circle.js"

            const myCircle = new Circle(2, 3, 1.5);

            if(myCircle.area() > UNIT_CIRCLE_AREA) {
                console.log("myCircle plus grand que le cercle unité");
            }

            import { UNIT_CIRCLE_PERIMETER } from "./Circle.js" // Erreur ! Car non exporté

Les imports seront indiqués entre { et }, même si un seul élément est importé

On ne peut importer des éléments que dans un module. En utilisant ce principe tout notre code sera un ensemble de modules.

JavaScript

Les modules

Pour chaque module, il est possible d'indiquer un export par défaut


            // Fichier Circle.js    
            export default class Circle{ ... } // Export par défaut
            export const UNIT_CIRCLE = new Circle(0, 0, 1);

L'export par défaut peut être importé en lui donnant n'importe quel nom, sans accolades. JavaScript sait qu'un import sans accolades est forcément l'export par défaut du module. 


            // Fichier main.js    
            import Circle from "./Circle.js" // Valide
            const circle = new Circle(1,2,3);
        

            // Fichier main.js
            import Cercle from "./Circle.js" // Également valide, ici Circle est appelé Cercle
            const circle = new Cercle(1,2,3);

JavaScript

Les modules

Il est possible de renommer un import avec le mot-clé as. Cela est particulièrement utile quand on importe deux éléments dont les noms sont les mêmes.


        import { Circle as Cercle, UNIT_CIRCLE as CERCLE_UNITE} from "./Circle.js"
C'est typiquement le cas quand on utilise plusieurs librairies différentes dans un seul fichier 

        import { Color as ColorLib1} from "./my/library1/Color.js"
        import { Color as ColorLib2} from "./my/library2/Color.js"

JavaScript

Les modules

Il est possible de mélanger les imports et les imports par défaut


        // Importe l'import pas défaut sous le nom de "Cercle", 
        // et l'import "UNIT_CIRCLE" sous le nom de "CERCLE_UNITE"
        import Cercle, {UNIT_CIRCLE as CERCLE_UNITE} from "./Circle.js"

Conseil : ne renommer les imports que lorsque cela est nécessaire, pas pour faire joli

JavaScript

Les modules

Pour importer un module JavaScript dans du HTML, on utilise la balise <script> avec type="module"


        <script type="module">
            import Circle from "./Circle.js";
            /* ... */
        </script>

Attention : l'isolation vaut toujours dans ce cas, et le contenu de la balise script sera isolé du reste du code


        <script>
            const value = 10;
        </script>
        <script type="module">
            console.log(value); // value n'existe pas ici, affichera undefined
        </script>
        <script>
            console.log(value); // affichera 10
        </script>
