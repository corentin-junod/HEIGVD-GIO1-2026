L'objectif

Cela permet

  • aux autres d'utiliser notre code
  • à sois-même, lorsqu'on aura besoin de le modifier
  • d'éviter des erreurs de logique et de bugs
  • d'être plus efficace dans le futur, pour ajouter des fonctions ou corriger des bugs

On produit du code de qualité pour les humains, pas pour la machine

Ces raisons sont suffisantes pour toujours produire du code respectant les standards de qualité. Un code difficile à comprendre sera souvent ignoré, lui faisant perdre beaucoup de son intérêt.

Programmation

La qualité



« Rendez les choses aussi simples que possible, mais pas plus simples. »
Albert Einstein



Plus concrètement :
Tout code doit être spontanément évidemment juste

Au delà des belles phrase, l'objectif final d'un code de qualité est d'être facilement compréhensible par les humains. En particulier, on ne cherchera pas à optimiser ou anticiper plus que nécessaire. Un bon code doit être aussi simple et direct que possible, l'intention doit être claire et il doit être évident que le code accomplit ce qu'il est censé faire sans erreur. Il se trouve que cela est plus difficile à faire qu'il n'y parait... On connaît généralement ce problème sous l'acronyme K.I.S.S : Keep It Simple, Stupid

	def get_x_y_centered(xs, ys, side, min_y_extent):
		if len(xs) <= 0: 
			return np.empty(0)
		x_min, y_min = np.min(xs), np.min(ys)
		x_extent = np.max(xs) - x_min
		y_extent = max(np.max(ys) - y_min, min_y_extent)
		resolution = x_extent / (side - 1) if x_extent > y_extent else y_extent / (side - 1)
		if resolution <= 0: 
			return np.empty(0)
		xs_centered = ((xs - x_min + (side * resolution - x_extent) / 2.0) / resolution)
		ys_centered = ((ys - y_min + (side * resolution - y_extent) / 2.0) / (-resolution))
		return np.column_stack((xs_centered, ys_centered)).astype(np.int32)

Quelques conseils généraux

  1. Respecter les standards de notation (du langage et du projet sur lequel vous travaillez). Exemples :
    • Les instances d'objets en minuscules : const monRectangle = ....
    • Les classes avec une majuscule : const monRectangle = new Rectangle()
    • Les constantes globales tout en majuscules : const NB_DIMENSIONS = 3
  2. Restez cohérents, utilisez les mêmes noms pour de mêmes choses. Exemples :
    • Choisissez entre camelCase ou snake_case
    • Si vous nommez une classe "Disk", utilisez le mot "disk" partout, et ne mélangez pas avec "Circle"
    • Prenez le temps avant de choisir un mot. Il existe souvent un mot exact pour ce que l'on veut.
      Par exemple, préférez isRetired plutôt que isOld
    • Utilisez des termes en anglais

L'optimisation


La loi d'Amdahl :

The overall performance improvement gained by optimizing a single part of a system is limited by the fraction of time that the improved part is actually used


Autrement dit : Il est peu utile d'optimiser une partie du code qui ne prend qu'une petite fraction du temps total d'exécution


Il faut éviter les optimisations prématurées ("premature optimization is the root of all evil", Donald Knuth)


La seule méthode efficace : mesurer les temps d'exécution et optimiser en fonction
