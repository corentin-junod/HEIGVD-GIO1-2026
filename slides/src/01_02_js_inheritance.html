Supposons avoir une classe Date


        class Date {
            constructor(year, month, day){
                this.year = year;
                this.month = month;
                this.day = day;
            }
            getDate(){
                return this.day+"."+this.month+"."+this.year;
            }
        }

        const date = new Date(2026, 10, 15);
        console.log(date.getDate()); // 15.10.2026

On souhaite maintenant créer une deuxième classe DateTime, stockant en plus l'heure. Il serait bien de pouvoir réutiliser la classe Date, sans devoir réimplémenter tout son contenu.

JavaScript

L'héritage

JavaScript permet de le faire de la manière suivante


        class DateTime extends Date {    // Le mot clé "extends" indique l'héritage
            constructor(year, month, day, hours, minutes, seconds){
                super(year, month, day) // "super" est le constructeur de la classe parente
                this.hours = hours;
                this.minutes = minutes;
                this.seconds = seconds;
            }
            getTime(){
                return this.hours+":"+this.minutes+":"+this.seconds;
            }
        }

        const dateTime = new DateTime(2023, 12, 23, 12, 30, 45);
        console.log(dateTime.getDate()); // 23.12.2023
        console.log(dateTime.getTime()); // 12:30:45

Supposons avoir une méthode print dans la classe Date.


        class Date {
            constructor(year, month, day){
                this.year = year;
                this.month = month;
                this.day = day;
            }
            getDate(){
                return this.day+"."+this.month+"."+this.year;
            }
            print(){
                console.log("Nous sommes le "+this.getDate());
            }
        }

        const date = new Date(2026, 10, 15);
        date.print(); // Nous sommes le 15.10.2026

Il va falloir modifier cette méthode dans la classe enfant DateTime.

L'héritage

On peut écraser (override) des méthodes venues des classes parentes.


        class DateTime extends Date {    // Le mot clé "extends" indique l'héritage
            constructor(year, month, day, hours, minutes, seconds){
                super(year, month, day) // "super" est le constructeur de la classe parente
                this.hours = hours;
                this.minutes = minutes;
                this.seconds = seconds;
            }
            getTime(){
                return this.hours+":"+this.minutes+":"+this.seconds;
            }
            print(){
                console.log("Nous sommes le "+this.getDate()+" à "+this.getTime());
            }
        }
        const dateTime = new DateTime(2023, 12, 23, 12, 30, 45);
        dateTime.print(); // Nous sommes le 23.12.2023 à 12:30:45

Comment choisir entre héritage et attribut ?

X sera un attribut de Y si on peut dire "Un Y possède un X"

X sera un parent de Y si on peut dire "Un Y est un X"


                    class Moteur{

                    }
                    class Voiture extends Moteur {   
                        constructor(){/*...*/}
                        break(){/*...*/}
                    }
                

                    class Moteur{
                        constructor(){/*...*/}
                        start(){/*...*/}
                        stop(){/*...*/}
                    }
                    class Voiture {   
                        constructor(){
                            this.motor = new Moteur();
                            /*...*/
                        }
                        break(){/*...*/}
                    }

L'idée de droite est la bonne : Une voiture possède un moteur, une voiture n'est pas un moteur
