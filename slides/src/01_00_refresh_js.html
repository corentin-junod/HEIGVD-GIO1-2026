Les fonctions

Les fonctions se déclarent avec le mot-clé function 


            function add(a, b){
                return a+b;
            }
            console.log(add(2,3)); // 5

En JavaScript, les fonctions sont des objets, qui peuvent être stockés dans des variables 


            const add = function (a, b){
                return a+b;
            };
            console.log(add(2,3)); // 5

On les appelle alors fonctions anonymes, car elles peuvent ne pas porter de nom.

En JS, les fonctions peuvent être déclarées en dessous de leur premier appel, cela n'a pas d'importance. On appelle cette propriété le hoisting.

JavaScript

Les fonctions

Les fonctions anonymes permettent de passer des fonctions en paramètre d'autres fonctions 


            const mul2 = function (a){return 2*a;}
            const squared = function (a){return a*a;}

            function apply(array, func){
                const result = [];
                for(const x of array){
                    result.push(func(x));
                }
                return result;
            }

            console.log(apply([1,2,3,4], mul2)) // 2,4,6,8
            console.log(apply([1,2,3,4], squared)) // 1,4,9,16
Utiliser la notation mul2 indique d'utiliser la fonction elle-même
Utiliser la notation mul2(x) appelle (execute) la fonction avec la valeur x

Les fonctions

Les fonctions fléchées (arrow functions) sont une notation encore plus compacte 


            const add = (a, b) => {return a+b;};
            console.log(add(2,3)); // 5

Si la fonction n'est composée que d'un return, on peut omettre les accolades et le return : 


            const add = (a, b) => a+b;
            console.log(add(2,3)); // 5

Toutes ces formes sont identiques, à l'exception de détails que nous n'aborderons pas dans ce cours

JavaScript

Les fonctions

Les fonctions fléchées sont très utilisées en JavaScript, elles simplifient beaucoup l'écriture des fonctions. 


            function apply(array, func){
                const result = [];
                for(const x of array){
                    result.push(func(x));
                }
                return result;
            }

            console.log(apply([1,2,3,4], a => 2*a)) // 2,4,6,8
            console.log(apply([1,2,3,4], a => a*a)) // 1,4,9,16

En pratique, les tableaux possèdent une méthode map qui fait exactement l'effet de apply 


            console.log( [1,2,3,4].map(x => 2*x) ) // 2,4,6,8
            console.log( [1,2,3,4].map(x => x*x) ) // 1,4,9,16
Si la fonction fléchée n'a qu'un paramètre, on peut même retirer les parenthèses autour du paramètre, comme dans l'exemple de cette slide.

Les objets

En JavaScript un objet est un ensemble de propriétés qui possèdent une clé et une valeur


        const myObject = {
            cle1 : "val1",
            cle2 : 35,
            cle3 : x => 2*x
        }

Il s'agit de l'équivalent des dictionnaires en Python

JavaScript

Les objets

Les clés peuvent être lues ou modifiées avec une notation utilisant un point ou des crochets


        const myObject = {
            nombre: 8
        };
        
        console.log(myObject.nombre);       // 8
        console.log(myObject['nombre']);    // 8

        myObject.nombre = 9;                // Assigne 9
        myObject['nombre'] = 9;             // Identique à la ligne précédente

Lire une clé qui n'existe pas retourne la valeur undefined

Assigner une clé qui n'existe pas crée cette clé

Les objets

On peut définir des fonctions dans un objet de deux manières


        const mesFonctions = {
            mult2 : function(x){           // Manière 1 : Assigner une fonction à une propriété
                return x*2;
            },
            mult3(x) {                     // Manière 2 : Directement écrire la fonction
                return x*3;                //             sans le mot-clé function
            }
        };
        console.log(mesFonctions.mult2(3)) // 6
        console.log(mesFonctions.mult3(3)) // 9

Ces deux manières sont équivalentes

L'orienté objet et les classes

JavaScript permet de créer des constructeurs. Un constructeur est une fonction qui crée des objets


        function Point(x,y){            // Point est un constructeur
            this.x = x;
            this.y = y;
            this.display = () => console.log("("+this.x+";"+this.y+")");
        }

        const p = new Point(10, 20);    // Crée un objet nommé "p"
        console.log(p.x);               // 10
        p.display()                     // (10;20)

Le mot clé this dans le constructeur permet d'assigner une propriété à l'objet qui sera créé

Le mot clé new indique de créer un objet en utilisant le constructeur

JavaScript

L'orienté objet et les classes

En JavaScript moderne, il existe un syntaxe plus proche des autres languages orientés objet.
Nous utiliserons cette syntaxe, plus lisible pour des objets complexes 


        class Point{
            constructor(x,y){   // Le constructeur est une fonction qui se nomme toujours "constructor"
                this.x = x;     // C'est cette fonction qui sera appelée avec "new"
                this.y = y;
            }
            display(){
                console.log("("+this.x+";"+this.y+")");
            }
        }
        
        const p = new Point(10, 20);    // Crée un objet nommé "p"
        console.log(p.x);               // 10
        p.display()                     // (10;20)

Les syntaxes de cette slides et la précédente sont équivalentes

On dira "classe" pour parler du type, mais pas d'un objet précis. Ainsi la "classe" Point possède les attributs x, y et la méthode display. On dira "objet" pour parler d'une instance de la classe. Ainsi la constance p est un objet de type Point, c'est-à-dire qui est une instance de la classe Point. Par convention, on commencera le nom des classes par une majuscule (Point) et on commencera le nom des objets par une minuscule (p).
